2. What is Git

Git is a popular version control system. It was created by Linus Torvalds in 2005, and has been maintained by Junio Hamano since then.
It is used for:

  • Tracking code changes
  • Tracking who made changes
  • Coding collaboration

It is used for:

  • Manage projects with Repositories
  • Clone a project to work on a local copy
  • Control and track changes with Staging and Committing
  • Branch and Merge to allow for work on different parts and versions of a project
  • Pull the latest version of the project to a local copy
  • Push local updates to the main project

Other Version control systems are:

  • Subversion (SVN): A centralized version control system that has been popular for many years.
  • Mercurial: A distributed version control system that is known for its simplicity and speed.
  • Bazaar: A distributed version control system that is designed to be easy to use and flexible.

8. Branches

In Git, a branch is a new/separate version of the main repository

Branches allow you to work on different parts of a project without impacting the main branch.

Why are they important?

  • Isolation: Work on new features without affecting the main code.
  • Parallel Development: Multiple people can work on different parts at the same time.
  • Experimentation: Try new ideas without breaking the main code.
  • Feature Flags: Control when new features are available.
  • Rollback: Go back to a previous version if something goes wrong.
  • Code Review: Check changes before merging them into the main code.
  • Release Management: Manage different versions of your software.
